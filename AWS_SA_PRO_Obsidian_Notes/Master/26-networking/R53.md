**6. [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|Route 53]] (R53) & Hybrid DNS**

Since on-premises servers cannot reach the AWS Resolver (10.0.0.2) and AWS cannot reach on-prem DNS servers by default, you must build a “bridge” using 

**[[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|Route 53]] Resolver Endpoints**:

- **Inbound Endpoint:**

  - What it is: Elastic Network Interfaces (ENIs) in your [[AWS_SA_PRO_Obsidian_Notes/Master/VPC|VPC]].

    - Flow: On-prem servers send DNS queries (via VPN/Direct Connect) to these ENI IP addresses.

    - Use Case: Allows on-prem servers to resolve AWS [[route53|Private Hosted Zones]] (e.g., app.aws.corp).

- **Outbound Endpoint:**

  - What it is: ENIs in your [[AWS_SA_PRO_Obsidian_Notes/Master/VPC|VPC]] used as egress points for DNS queries.

  - Flow: The AWS Resolver sends queries _out_ through these ENIs to your on-prem DNS server IPs.

  - Use Case: Allows [[ec2]] instances to resolve on-prem domains (e.g., legacy.corp).

**7. R53 CNAME vs. Alias**

- **CNAME (Canonical Name):**

  - Standard DNS record. Maps www.example.com -> lb-123.aws.com.

  - Restriction: **Cannot** be used at the “Zone Apex” (the root domain, like example.com). You can only use it for subdomains.

- **Alias Record:**

  - AWS-specific extension to DNS.

  - Superpower: Can be used at the Zone Apex (example.com).

  - Function: Maps a domain directly to an AWS resource (Load Balancer, [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|CloudFront]], [[api-gateway|API Gateway]], [[AWS_SA_PRO_Obsidian_Notes/Master/VPC|VPC]] Endpoint).

  - Cost: [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|Route 53]] does **not charge** for queries to Alias records that point to AWS resources, whereas CNAME queries are billable.

**8. Gateway vs. [[AWS_SA_PRO_Obsidian_Notes/Master/03-networking/privatelink|Interface Endpoints]]**

These are the two types of **[[AWS_SA_PRO_Obsidian_Notes/Master/03-networking/privatelink|VPC Endpoints]]** used to access AWS services privately.

- **[[AWS_SA_PRO_Obsidian_Notes/Master/03-networking/privatelink|Gateway Endpoints]]:**

  - Services: Only supports **Amazon [[AWS_SA_PRO_Obsidian_Notes/Master/S3|S3]]** and **Amazon [[dynamodb]]**.

  - Mechanism: It works by adding a route to your [[AWS_SA_PRO_Obsidian_Notes/Master/VPC|VPC]] Route Table (e.g., Dest: pl-63a540 / Target: vpce-123).

  - Cost: Free.

- **Sharing:** These rules are regional. To use them across a multi-account organisation, you share the rule via [[ram]]. The association of a rule with a [[AWS_SA_PRO_Obsidian_Notes/Master/VPC|VPC]] triggers the AWS Resolver in that [[AWS_SA_PRO_Obsidian_Notes/Master/VPC|VPC]] to start using that logic.
