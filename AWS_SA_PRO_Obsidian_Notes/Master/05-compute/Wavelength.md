**Advanced Architecture**

Wavelength is a part of AWS's Local Zones and Outposts services, providing single-digit millisecond latencies to 5G devices by extending AWS infrastructure to the edge of the network. Wavelength zones are located in or near the metro areas served by 5G networks. They support AWS mobile servers like [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|App Runner]], [[ecs]], [[eks]], and [[lambda]].

[[RDS_Instance_Types|Internals]]: Wavelength uses a modified version of NITRO system, allowing it to integrate with the underlying 5G network via the Wavelength Converged Edge (WCE). WCE enables direct communication between Wavelength and the 5G network core, minimizing latency.

[[RDS_Instance_Types|Global scale considerations]]: Wavelength supports multi-account strategies using AWS [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|Resource Access Manager (RAM)]] and [[organizations|AWS Organizations]]. RAMP allows easy sharing of resources across accounts without replicating them. [[ram]] can share [[transit-gateway|AWS Transit Gateway]] Network Manager resources, simplifying management of Wavelength VPCs.

**Comparison & Anti-Patterns**

| Service          | Use Case                                                                   |
| ---------------- | ------------------------------------------------------------------------- |
| Wavelength       | Mobile applications requiring ultra-low latency (<10 ms)                |
| Local Zone       | Applications needing low latency (10-20 ms) but not compatible with 5G |
| Outposts        | Applications requiring on-premises AWS infrastructure                      |
| **Anti-pattern:** | **Use Case**                                                              |
| Wavelength     | Non-mobile applications, non-5G supported regions                         |
| Local Zone    | Highly sensitive data that cannot be hosted outside primary region         |
| Outposts       | Low-latency requirements met by Wavelength or Local Zone                 |

**Performance & Reliability**

Throttling limits: Wavelength throttles API requests based on AWS service limits. To manage throttling, implement exponential backoff strategies to retry failed API calls.

HA/DR patterns: For high availability, distribute workloads across multiple Wavelength zones within a region. For [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|disaster recovery]], implement backup and restore solutions using [[Master/Git_hub_notes/AWS-SAP-C02-Notes-main/README|AWS Backup]] or custom scripts.

**Professional Exam Scenarios**

Scenario 1: A media company wants to deliver live sports content with sub-10ms latency to users connected to 5G networks. The application requires automatic scaling and minimal server maintenance. Which solution should they choose?

Correct answer: Wavelength with [[lambda|AWS Lambda]]. This solution provides ultra-low latency, automatic scaling, and minimal server maintenance.

Incorrect answer: Using Local Zone would increase latency beyond the required limit. Outposts do not provide the necessary low-latency benefits.

Scenario 2: An [[iot]] device manufacturer needs to process data generated by devices with sub-10ms latency. They want to minimize operational overhead while maintaining control over their infrastructure.

Correct answer: Wavelength with AWS [[Fargate]]. This solution enables low-latency processing, reduces operational overhead, and maintains infrastructure control.

Incorrect answer: Using Local Zone would increase latency beyond the required limit. Outposts require manual maintenance, which goes against the requirement.
